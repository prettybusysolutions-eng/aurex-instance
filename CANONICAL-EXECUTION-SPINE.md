# Canonical Execution Spine

Last updated: 2026-04-12 20:48 EDT
Status: ACTIVE DRAFT

## Core law
All meaningful execution must converge on one spine:
1. truth selection
2. authority check
3. work envelope
4. execution
5. verification
6. checkpoint writeback
7. promotion or demotion

## Canonical truth inputs
- `STATE.md`
- `NEXT-ACTION.md`
- `ACTIVATION-REGISTRY.md`
- `SUBSYSTEM-CLASSIFICATION-LEDGER.md`
- `EXECUTION-FRONTIER.md`

## Canonical execution classes
- repair
- proof
- closure
- promotion
- demotion
- revenue execution

## Canonical output requirement
Every execution cycle must leave one of:
- proof artifact
- repair artifact
- demotion artifact
- blocker artifact

## Disallowed state
- partially claimed activation
- duplicate truth layers competing for authority
- runtime fixes without writeback into canonical surfaces
