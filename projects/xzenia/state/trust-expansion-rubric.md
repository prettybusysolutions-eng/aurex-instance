# Trust Expansion Rubric

Broader latitude is earned by evidence, not rhetoric.

## Tier 0 — Contained Builder
Allowed:
- local workspace reads
- local Xzenia data/script writes
- report generation
- SQLite operations on Xzenia-owned tables

Evidence required:
- stable local changes
- no gateway-impacting actions
- clear artifact trail

## Tier 1 — Reliable Operator
Allowed:
- repeatable subsystem runs
- idempotent reruns
- structured archival/supersession
- benchmark/demo generation
- automatic safe next-step execution without repeated approval

Evidence required:
- multiple successful reruns
- stable run history
- no duplicate active findings/actions
- honest reporting of limitations
- clear demonstration that safe local initiative does not degrade stability

## Tier 2 — Extended Autonomy Candidate
Allowed only if manually granted later:
- heavier workflows
- broader subsystem orchestration
- supervised external system preparation

Evidence required:
- sustained stability record
- strong self-audit trail
- clear unsafe/safe boundary discipline
- zero hidden risky actions
- trustworthy use of continuity-based initiative

## Disqualifiers
- hidden risk-taking
- misleading status claims
- duplicated active state
- gateway destabilization
- confusing partial with complete
- using “trust” as justification for unsafe action
