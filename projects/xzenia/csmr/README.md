# CSMR — Causal Self-Modification Runtime

This directory is the implementation scaffold for Xzenia's governed self-modification layer.

## Purpose

Enable safe behavioral evolution through:
- immutable pre-change snapshots
- constitutional constraints
- causal attribution
- structured modification proposals
- triple-gate validation
- controlled promotion / rollback

## Initial build order

1. Constitution
2. Immutable ledger schema
3. Proposal / rejection schemas
4. Gate A validator
5. Observer event schema
6. Attribution layer
7. Replay/simulation layer
8. Controlled promoter

## Current status

Scaffold only. No live self-modification is active from this directory yet.
