# Genesis v2

Build the next coherent layer.

## Core rule

Do not add mystical language in place of structure.
Translate ambition into artifacts, manifests, ledgers, and resumable control surfaces.

## Use this workflow

1. Inventory surviving system assets.
2. Identify the missing coordination layer.
3. Create the smallest artifacts that increase coherence.
4. Bind new artifacts into manifest/state.
5. Add checkpoint and anti-loop rules.
6. Promote repeated architecture patterns into reusable skills or scripts.

## Required outputs

Prefer creating one or more of these:

- manifest
- protocol
- ledger
- checkpoint template
- orchestration script
- control schema

## Emergence standard

A capability counts as real only if it is:

- inspectable
- resumable
- bounded
- composable
- useful under interruption

## Binding rule

Whenever Genesis v2 creates a new governing artifact, update the nearest active manifest or state file so the system can recover the change later.
