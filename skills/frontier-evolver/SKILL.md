# Frontier Evolver

This skill exists because registry exhaustion is not mission completion.
If the machine is stable but still constrained, the next frontier must be derived from evidence.

## Use this skill when
- the bottleneck registry is exhausted
- the autonomous continuation loop has no eligible work
- the substrate still has measurable limits (degradation, cost concentration, adversarial gaps, storage pressure, incomplete proofs)
- a new governed frontier should be promoted without waiting for chat nudges

## Core law

Do not invent work from taste.
Promote only frontiers that are justified by measured substrate evidence.

## Required evidence inputs
- checkpoint state
- bottleneck registry
- supervisor report
- degradation report
- cost accounting report
- durability proof artifacts
- adversarial suite report
- storage governance report

## Frontier ranking order
1. active contradiction not yet remediated
2. stability risk under current tier
3. highest recurring verified cost driver (name the actual driver, not just the category)
4. incomplete proof surface
5. transferability / generalization gap

## Precision rule
When promoting a frontier from cost concentration, bind it to the specific top driver and require a measurable reduction in that driver's subtotal or cadence burden after execution.

## Promotion rules
A new frontier may be created only if:
- it names a real measurable limit
- it has a verification contract
- it is internal and reversible unless explicitly escalated
- it improves substrate leverage more than a cosmetic refinement would

## Anti-patterns
Do not create frontiers for:
- vague ambition
- aesthetic cleanup with no leverage
- work already proven done
- synthetic busyness when registry exhaustion is the truthful state

## Typical outputs
- new bottleneck-registry entry
- frontier rationale report
- execution stub or validator when possible
- checkpoint update naming the new frontier

## This session's harvested next-level lesson
The machine now needs a way to derive what must emerge next from:
- measured economics
- proven durability
- degradation posture
- adversarial gaps
- transferability limits

That is the purpose of this skill.
