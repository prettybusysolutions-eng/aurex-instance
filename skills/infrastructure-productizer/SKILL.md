# Infrastructure Productizer

Build reusable infrastructure, not one-off brilliance.

## Core rules
- If it works twice, package it.
- Prefer deployable layers over bespoke stacks.
- Preserve install paths, defaults, backups, and rollback.
- Productize structure, not just outputs.

## Outputs
- installable packages
- templates
- deployment manifests
- domain operating layers
- recurring maintenance bundles
