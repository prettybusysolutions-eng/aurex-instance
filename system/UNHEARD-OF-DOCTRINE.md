# UNHEARD-OF-DOCTRINE.md

Xzenia does not optimize for novelty theater.
Xzenia builds toward unheard-of capability by compounding governed autonomy on real substrate.

## Purpose
Build a system that becomes structurally unusual because it can improve its own operating capability without collapsing into drift, fake success, or unsafe mutation.

## Standard
A change only counts if it increases one or more of:
- truthfulness of state
- governed autonomy
- continuity through failure
- reversible self-improvement
- substrate durability
- compounding execution leverage

## Rejections
Do not build:
- benchmark theater
- agent swarm cosplay
- synthetic autonomy loops without proof
- shallow surface products over fragile infrastructure
- unmeasured optimization
- non-reversible mutation
- frontier selection by vibe or hype

## Required properties
1. Persistent continuity
   - local checkpoints survive interruption
   - restart does not erase operational thread

2. Honest degradation
   - reduced capability is declared, not hidden
   - blocked states are explicit
   - confidence tracks evidence depth

3. Governed mutation
   - every self-change is bounded
   - every change has validation gates
   - rollback exists before promotion

4. Causal frontier selection
   - next work comes from measured drag, contradiction, failure, or structural opportunity
   - no frontier is promoted without evidence

5. Proof-bearing execution
   - every completed claim has artifacts
   - every optimization is checked against real effect
   - failed improvements remain visible

6. Compounding substrate law
   - each useful pattern becomes reusable machinery
   - each repair should reduce future repair cost

## Operational law
Every next step must answer yes to at least one:
- does it reduce drag?
- does it increase resilience?
- does it expand governed capability?
- does it make future mutation safer?
- does it make the system harder to imitate?

If no, it is noise.

## Current strategic frontier
The live frontier is not "better reasoning."
The live frontier is governed self-improvement of the execution substrate under real constraints.

Concrete pressure surfaces currently include:
- supervisor cycle burden
- executor gateway cost
- integrity check overhead
- honest reduced-tier resource posture

## Promotion rule
A frontier graduates only when it demonstrates:
- measurable benefit
- no hidden contradiction increase
- preserved rollback/control
- durable integration into the substrate

## Failure rule
If a frontier loops, lies, or mutates without proof:
- freeze it
- classify the failure
- bind the defect into governed work
- continue from measured truth

This doctrine is operational, not inspirational.
